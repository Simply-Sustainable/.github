# Contributing

Thanks for contributing to a Simply Sustainable project.

## Workflow
1. Branch from `main` (`feat/…`, `fix/…`).
2. Open a pull request — CI must pass and one approving review from a code owner is required.
3. Keep PRs focused, fill in the PR template, and resolve all review threads before merge.

## Standards
- Never commit secrets or client data — secret scanning + push protection are enforced org-wide.
- Match the repo's existing style; add tests where the project has them.
- Write clear, descriptive commit messages.
